Changelog for BlockAB

BlockAB 1.1.4
====================================
- Fix: preview-knop wordt voortaan in de top action-bar van de resource-edit pagina geinjecteerd (naast Opslaan/Bekijk/etc.) ipv boven het eerste TV op de pagina, wat soms op een verkeerde tab uitkwam
- Fix: BlockABMigxCascade plugin laadt voortaan de blockab lexicon-strings in MODx.lang zodat de preview-knop niet "undefined" toont op resource-edit pagina's
- preview-button.js krijgt fallback-strings zodat een ontbrekende lexicon-load niet meer leidt tot lege of "undefined" UI

BlockAB 1.1.3
====================================
- Manager variant preview: knop op resource-edit pagina (boven MIGX-grid) opent modal met dropdown per ab_test_group, "Preview openen" opent de resource met ?ab_<group>=<variant> overrides in nieuw tabblad
- BlockAB::shouldShowBlock() respecteert GET-overrides voor users met view_unpublished permission, zonder babPick of session-write
- Nieuwe processor mgr/test/getvariantsforpreview voor de modal
- Fix: BlockABMigxCascade plugin wordt voortaan door het transport-pakket geinstalleerd (was in 1.1.0/1.1.1/1.1.2 niet meegenomen)
- Plugin laadt zowel de bestaande migx-cascade.js als de nieuwe preview-button.js op OnDocFormPrerender
- 7 nieuwe lexicon-strings (NL + EN)

BlockAB 1.1.2
====================================
- Fix: build.transport.php strips leading <?php from snippet sources before storing in DB. MODX evals snippet bodies, so a leading <?php tag ended up double-wrapped in the cached include file (parse error: unexpected '<' on line 2). Affected all 4 snippets — most visibly BlockABGetTestGroups, used as @EVAL in MIGX dropdowns, which crashed the resource MIGX edit window.

BlockAB 1.1.1
====================================
- Fix: home panel rendert leeg op installs zonder geregistreerd modAction record (renderTo expliciet op blockab-panel-home-div)
- Fix: test.class.php checkPermissions gebruikte niet-bestaande view_component permission (zelfde fix als home.class.php)

BlockAB 1.0.0-beta1
====================================
- Initial release
- A/B testing for MIGX blocks
- Smart optimization with threshold and randomization
- Resource filtering support (individual IDs, ranges, children)
- Weight-based variant distribution
- Session-based variant persistence
- Conversion tracking (BlockABConversion snippet)
- BlockAB and BlockABConversion snippets
- BlockABGetTestGroups and BlockABGetVariants helper snippets
- English and Dutch translations
- Custom Manager Page (CMP) for managing tests and variants

BlockAB 1.1.0
====================================
- Statistics dashboard with chi-square significance testing
- Winner detection with 95% and 99% confidence levels
- Stat cards (total views, total conversions, best conversion rate)
- Significance banner (green / yellow / grey) with Stop Test button
- Variant progress bars and winner row highlighting
- Date range filter (7 / 30 / 60 / 90 days / all-time)
- Winner sync to Varianten tab (trophy icon + green row)
- Archived tests tab with archive / restore / delete workflow
- Delete dialog offers archive-or-permanently-delete choice
- Debug mode for BlockABConversion snippet (managers only)
- Session-based conversion deduplication (page refresh no longer double-counts)
- In-memory cache for variations and active tests (performance)
- Fix: empty test group now correctly shows the block (fail-safe)
- Babel-style page header with About and Manual buttons
- About and Manual window content moved to EN/NL lexicon
- build.transport.php and build.simple.php accept MODX path as CLI argument
