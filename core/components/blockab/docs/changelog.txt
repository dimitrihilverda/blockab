Changelog for BlockAB

BlockAB 1.1.11
====================================
- Hover menu rebuilds its items on every show, so dirty-state and MIGX block changes (e.g. adding a second variant to a block, or saving a single block via the MIGX edit modal) are reflected immediately without needing a full resource save first.
- Subtle dirty-state notice (⚠ + qtip with full sentence) appears at the top of the hover menu when the resource has unsaved changes — same information as the modal warning, but compact.
- Added blockab.preview_dirty_short lexicon string (NL + EN).

BlockAB 1.1.10
====================================
- Polish: variant submenu and combine modal now only list variants that are actually placed in the resource's MIGX blocks. If the test defines A/B/C/D but the resource only uses A and B, only A and B appear as preview options. Same filter applies to the combine modal dropdowns.

BlockAB 1.1.9
====================================
- Polish: "Combineer varianten..." menu item now only appears when the resource has 2 or more ab_test_groups. With a single group there is nothing to combine, so the divider + item are hidden.

BlockAB 1.1.8
====================================
- Fix: preview button now reappears correctly after the resource is saved. Previous logic relied on a sticky buttonInjected flag and a 10-second MutationObserver disconnect; if MODX rebuilt the toolbar (post-save) or MIGX rendered later than 10s the button stayed gone. New logic checks each iteration whether the button is actually attached to the current modx-action-buttons toolbar, throttles DOM-mutation reactions to 200ms, and keeps the observer running for the lifetime of the page.

BlockAB 1.1.7
====================================
- Fix: visits by users with an active manager session no longer write to babPick or babConversion. Manager preview / QA / editors visiting their own thank-you page must not pollute test stats. Gate is hasSessionContext('mgr'), checked in pickOne() (skips registerPick) and at the top of registerConversion() (early return with debug log line "SKIP: ingelogde manager").

BlockAB 1.1.6
====================================
- Preview button is now a proper Ext hover-menu (same UX as Babel's language switcher): hover the button → submenu per ab_test_group → click a variant to open the resource in a new tab with that single variant active
- Multiple ab_test_groups in one resource: each gets its own submenu, clicking a variant overrides only that group
- "Combineer varianten..." menu item still opens the form modal for picking a combination across multiple groups
- Combine modal got more horizontal room (620px wide, 240px label width) so longer test_group names fit comfortably
- 3 new lexicon strings (blockab.preview_combine, blockab.preview_loading, blockab.preview_no_variants — NL + EN)

BlockAB 1.1.5
====================================
- Fix: preview button is now added as an Ext.Button to the modx-action-buttons toolbar (via toolbar.insertButton(0, [...]) + doLayout) instead of as raw HTML next to it. Same pattern as Babel's language switcher. The button now lines up correctly inside the toolbar regardless of theme or other plugins.

BlockAB 1.1.4
====================================
- Fix: preview button is now injected into the resource's top action-bar (next to Save / View / etc.) instead of above the first MIGX TV on the page — the previous placement could land on the wrong tab when a resource had multiple MIGX TVs
- Fix: BlockABMigxCascade plugin now loads the blockab lexicon strings into MODx.lang so the preview button renders its label instead of "undefined" on the resource edit page
- preview-button.js gets a _t(key, fallback) helper to keep the UI readable if the lexicon topic ever fails to load

BlockAB 1.1.3
====================================
- Manager variant preview: button on the resource edit page opens a modal with one dropdown per ab_test_group; "Open preview" opens the resource in a new tab with ?ab_<group>=<variant> overrides applied
- BlockAB::shouldShowBlock() honours GET overrides for users with the view_unpublished permission without writing a babPick or session entry, so manager previews don't pollute test stats
- New processor mgr/test/getvariantsforpreview backs the modal
- Fix: BlockABMigxCascade plugin is now installed by the transport package (was missing from 1.1.0 / 1.1.1 / 1.1.2 — only present in staging because it had been created manually)
- Plugin now loads both the existing migx-cascade.js and the new preview-button.js on OnDocFormPrerender
- 7 new lexicon strings (NL + EN)

BlockAB 1.1.2
====================================
- Fix: build.transport.php strips leading <?php from snippet sources before storing in DB. MODX evals snippet bodies, so a leading <?php tag ended up double-wrapped in the cached include file (parse error: unexpected '<' on line 2). Affected all 4 snippets — most visibly BlockABGetTestGroups, used as @EVAL in MIGX dropdowns, which crashed the resource MIGX edit window.

BlockAB 1.1.1
====================================
- Fix: home panel rendered empty on installs without a registered modAction record (added explicit renderTo on blockab-panel-home-div)
- Fix: test.class.php checkPermissions referenced the non-existent view_component permission (same fix as home.class.php)

BlockAB 1.0.0-beta1
====================================
- Initial release
- A/B testing for MIGX blocks
- Smart optimization with threshold and randomization
- Resource filtering support (individual IDs, ranges, children)
- Weight-based variant distribution
- Session-based variant persistence
- Conversion tracking (BlockABConversion snippet)
- BlockAB and BlockABConversion snippets
- BlockABGetTestGroups and BlockABGetVariants helper snippets
- English and Dutch translations
- Custom Manager Page (CMP) for managing tests and variants

BlockAB 1.1.0
====================================
- Statistics dashboard with chi-square significance testing
- Winner detection with 95% and 99% confidence levels
- Stat cards (total views, total conversions, best conversion rate)
- Significance banner (green / yellow / grey) with Stop Test button
- Variant progress bars and winner row highlighting
- Date range filter (7 / 30 / 60 / 90 days / all-time)
- Winner sync to Varianten tab (trophy icon + green row)
- Archived tests tab with archive / restore / delete workflow
- Delete dialog offers archive-or-permanently-delete choice
- Debug mode for BlockABConversion snippet (managers only)
- Session-based conversion deduplication (page refresh no longer double-counts)
- In-memory cache for variations and active tests (performance)
- Fix: empty test group now correctly shows the block (fail-safe)
- Babel-style page header with About and Manual buttons
- About and Manual window content moved to EN/NL lexicon
- build.transport.php and build.simple.php accept MODX path as CLI argument
